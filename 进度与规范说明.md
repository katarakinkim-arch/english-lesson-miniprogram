- 用户的死命令（必须一直遵守）：

## 2. 硬性规定（每课必须做到，否则不算完成）

1. **权威资料核实** —— 用 WebSearch 验证事实 / 教学招式，记录来源（谁说的、哪来的）。不得凭空编。
2. **真实教学招式提炼** —— 融合模板法：搜高赞 / 高收藏讲课视频 → 读逐字稿 → 聚合招牌教学招式 → 以最受欢迎的课为**模板融合**（学结构，不克隆原话，版权合规）。共识知识点 ≠ 模仿风格 ≠ 抄袭。
3. **自由授权真实照片** —— 来源 Wikimedia Commons，授权须为 **PD / CC BY / CC BY-SA**；有真照才用真照，无则**学科色块兜底并明确标注**，绝不糊弄。禁止使用带强制水印的 ImageGen 图。
4. **手写精排 PPTX** —— 9 页固定结构：
   `封面 / 目标 / 背景 / 重点 / 难点 / 探究 / 板书 / 作业 / 总结`
   设计系统（见 `tools/_classroom_lib.py`）：暖纸 `#F4EFE6` / 墨蓝 INK / 霜红 FROST / 湘碧 XIANG / 金 GOLD / 灰 MUTED；楷体 KAI + 微软雅黑 HEI；16:9（13.333″×7.5″）。
5. **四层审计门禁全 PASS**（缺一不可，脚本在 `tools/`）：
   - `_audit_text.py` —— 教师口吻扫描（BLOCK / WARN / WHITELIST 三层）
   - `_check_bounds.py` —— shape 越界检查（限 13.333×7.5）
   - `_audit_scrim.py` —— 照片压文字页必须有遮罩保护
   - `_audit_layout.py` —— 文字框内溢出（>1.08×）+ 照片文字重叠
6. **纪律** —— 通篇**不出现「教师 / 老师」字样**（这是课堂学生版）；每页一个视觉焦点。

## 3. 关键坑位（复用必读，已经踩过并修过）

- **`scrim` 的 `a:alpha` 单位坑（必记）**：OOXML `a:alpha` 是千分之一百分比（100% = 100000）。旧写法 `int(alpha_pct*1000)` 会把 35% 写成 350 = 0.35% 不透明，遮罩近乎透明、对比度崩。**正确：`int(alpha_pct*100000)`**。6 处 scrim 定义全改过，预览器 `_pptx_to_png.py` 也加了 `a:alpha` 合成。
- **审计口吻泄漏**：`_audit_text` 不只扫 `process` 正文，**step_name / 封面 title / kicker 也要清洗**（曾因步骤名含「教师当场修改示范」触发 FAIL）。
- **黑板页字号**：`s_blackboard` 必须用闭环拟合字号（`_fit_blackboard`，复刻审计度量二分找最大字号），不能用「行数 × 固定字号」估高（会低估→溢出）。
- **`_check_bounds`**：查 shape 是否超 13.333×7.5；每页输出 `ok / OVERFLOW`。
- **Wikimedia 搜索 API 错配严重**：搜 "eagle"→白头海雕、"fish"→热带鱼。改用**维基百科条目 pageimages API** 取人工精选主图，或严格按内容关键词 + 人工核对。地理 / 内容必须对应篇目。
- **PIL `_audit_layout.est_text_h` 的 `\n` 空行 bug（已修）**：旧版把文本内每个 `\n` 当「换行 + 空行」，多行文本框高度被估成 2 倍 → 误报 TEXT_OVERFLOW。正确：单个 `\n` 只换行不加空行。映射器 `_classroom_ppt._est_text_h` 复刻必须与此一致（按行 split 后逐行 wrap，不补空行）。
- **盲 grep 教师口吻 = 假阴性**：PPTX 是 zip 压缩 XML，对二进制 grep 中文是假阴性，必须解压提取（用 `_audit_text.py`）。
- **照片复用**：同单元相邻课复用已下照片，避免重复下载。
- **`_audit_text` 的 WHITELIST**：部分中性词（如「本文」「全诗」）需进白名单，避免误杀；新增白名单要谨慎。

## 4. 在另一台电脑上怎么续做

### 4.1 环境与依赖
- Python 3.13+（建议 venv 隔离）
- 依赖：`Pillow`（PIL）、`python-pptx`
  ```bash
  python -m venv .venv
  source .venv/bin/activate        # Windows: .venv\Scripts\activate
  pip install pillow python-pptx
  ```

### 4.2 工具链位置（`tools/`）
| 文件 | 作用 |
|---|---|
| `_classroom_lib.py` | 共享设计库（颜色 / 字体 / 画框 / 遮罩 / 文本 / 板书等），所有渲染器复用 |
| `_classroom_ppt.py` | 逐课映射器（剥离教师口吻 + 9 页结构映射）|
| `_audit_text.py` / `_audit_scrim.py` / `_audit_layout.py` / `_check_bounds.py` | 四层审计门禁 |
| `_pptx_to_png.py` | 渲染预览（可出 PDF 便于浏览 / 打印）|
| `_render_u2_X.py` | 已完成的手写渲染器**范本**（照这个写新课）|
| `_photos_u2_X/` | 已下载的自由授权真实照片（续做同单元可复用）|
| `_build_standard_all.py` | 标准门禁批量管线（渲染→四层审计→溢出自动缩放重渲→留 JSONL）|

### 4.3 数据源
- 语文：`data/lessons-cn.js`（id 形如 `l-cn-bs-u{单元}-{序号}`，如 `l-cn-bs-u2-3`）
- 其他科：`data/lessons*.js`（英 / 政 / 史 / 地 / 生 / 数 / 物 / 化）

### 4.4 逐课标准流程（照做即可）
1. 读 `data/lessons-cn.js` 对应课数据（课题 / 重难点 / 教学过程）。
2. **WebSearch** 核实权威资料 + 提炼教学招式，把来源记下来（写进跟踪表）。
3. **Wikimedia Commons** 找自由授权真实照片，下载到 `tools/_photos_u{X}_{Y}/`。
4. 照 `tools/_render_u2_2.py` 范本写 `tools/_render_u{X}_{Y}.py`（手写 9 页，全程中性表述，无「教师 / 老师」）。
5. 跑四层审计（任意顺序）：
   ```bash
   python tools/_audit_text.py    ../preview_v7/cn/l-cn-bs-uX-Y.pptx
   python tools/_check_bounds.py  ../preview_v7/cn/l-cn-bs-uX-Y.pptx
   python tools/_audit_scrim.py   ../preview_v7/cn/l-cn-bs-uX-Y.pptx
   python tools/_audit_layout.py  ../preview_v7/cn/l-cn-bs-uX-Y.pptx
   ```
   四层必须全 PASS；若有 FAIL，回到第 4 步修渲染器重渲。
6. 全 PASS 后出预览：`python tools/_pptx_to_png.py ../preview_v7/cn/l-cn-bs-uX-Y.pptx ../preview_v7/cn/_preview_uX_Y/`。
7. 更新 `preview_v7/_RESEARCH_TRACK.md`（加一行：ID / 课题 / 来源 / 照片 / 审计 / 渲染器）+ 当日日志。
8. `git add` 相关文件 → `git commit` → `git push origin main`。

### 4.5 批量管线（省力但质量低于人工精调）
- `_build_standard_all.py` 可对全 866 课统一执行（渲染→四层审计→溢出自动缩放重渲→留 JSONL），适合铺量。
- **但「逐课调研」的人工环节（WebSearch + 真实照片 + 手写精排）无法被批量脚本替代**——857 课物理上不可能每课手写脚本。已完成的 5 课是人工精调样本；批量管线走「映射器 + 门禁」，质量低于这 5 课，且照片走 `_photo_map` CURATED（有真照则用，否则色块兜底）。

## 5. 诚实局限（必须知道，别被「看起来有文件」骗）

- 857 课做不到 857 次视频 / 图片调研；照片走 `_photo_map` CURATED，**非每课真实照**。此局限已向用户明示。
- 钟扬等人物肖像若涉版权（如新华社）未使用，以环境照替代并标注。
- 那 64 课旧产物（必修上全 8 单元）**不代表达标**——它们是色块兜底、无真实照片、无审计，按死命令要回炉。
- 当前 5 课是「逐课真实调研」标杆；其余走标准门禁管线，质量低于这 5 课的人工精调。
